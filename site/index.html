What Juno is — and isn't

The one thing the platforms won't build

Google, Meta, and OpenAI ship free native “is this AI?” detection into search and browsers. Out-detecting them is a losing game. Juno owns the niche they structurally will not: a neutral, cross-vendor, economically-liable record with persistent memory — one that remembers a piece of media was attested as AI even after the watermark or credential is stripped. It ingests many independent signals — C2PA, open SynthID-Text, watermark detectors, platform disclosures — so no single vendor is load-bearing.

Strong

Neutral registry

A multi-writer record of provenance attestations that no single AI firm owns or can censor.

Strong · scoped

Economic liability

Attesters stake; provably-wrong attestations are slashable. Liability covers detectable disagreement — not the truth.

Conditional

Bootstrap incentives

Emissions can subsidize coverage. If that doesn’t validate, Juno ships with stablecoin collateral and no token.

Why it’s different

It remembers after the watermark is gone

A credential can be stripped and a file re-encoded in seconds. Keyed by a perceptual fingerprint, the registry still surfaces the prior attestation — a labeled, dated prior, never a fresh verdict.

Only a neutral registry can do this. Google’s label is per-asset and ephemeral; a C2PA manifest is trivially stripped. The on-chain prior, keyed by fingerprint, survives both.

The trust primitive

A claim of record — not a verdict

Juno never asserts “this is AI.” It records a sourced, dated, re-runnable observation — which gives the slashing market deterministic ground truth and bounds defamation risk.

what we never say
“This image is AI-generated.”

A present-tense factual verdict — unprovable on-chain, and defamatory if wrong about a real person.

what we record
Attester A ran SynthID-Text v2 over this content on 2026-06-12 and observed a watermark.

A sourced, dated claim anyone can re-run and dispute. See JIP-4.

How it works

Fingerprint → attest → record → read → challenge

The chain stores signed claims and stake. The chain never runs detection — correctness is enforced after the fact by the economic-liability layer.

01

Fingerprint

Media is hashed client-side: chash (exact) + phash (near-dup). Raw media never leaves the device.

02

Attest

A staked oracle runs the signals (C2PA, open SynthID-Text, more) off-chain and signs a claim of record — which detector saw what.

03

Record

Written to the registry (EAS on an L2): chash dedup, staked-writer gating, near-dup linkage.

04

Read — free

Integrators query by fingerprint for the claim, confidence, lineage, and challenge status. Reads cost nothing.

05

Challenge

Anyone can dispute a wrong attestation within a window. Resolution slashes the loser and revokes the record.

$

Who pays?

Only fresh detection is paid — in stablecoin, by liability-bearing buyers who need a neutral record they can cite.

Built in the open

A real foundation, not a whitepaper

The canonical attestation schema (JIP-1) is an executable, test-verified artifact: one source of truth deriving the EAS schema string, the Solidity tuple, and a Borsh struct — with a round-trip equivalence test.

  • Testnet: Base Sepolia + EAS, a single permissioned attester. No federation in the MVP.
  • Mainnet: default stays on the L2; Solana state-compression is conditional.
  • Contracts: JunoAttestationModule (dedup + staked-writer gating) + StakingManager, with invariant tests.
  • Specs: four versioned Juno Improvement Proposals (JIPs).

We lead with our weak points

Three kill-gates

Before any token or mainnet spend, three questions must be answered. If any fails, Juno pivots to a stablecoin consortium registry — or stops. This is in the roadmap, not buried.

1

Does Google’s API even permit this?

The detection-API terms may forbid on-chain re-attestation, paid relay, or federation. A binary, potentially fatal, currently-unconfirmed legal question — diligence item #1.

2

Will anyone actually pay?

≥3 signed paid-pilot LOIs from liability-bearing buyers who can’t get the same data free from Google/C2PA — required before any token or mainnet spend.

3

Is the matching precise enough?

A published perceptual-hash precision number against a ≥1M hard-negative haystack. A false “AI” label is a defamation risk, so persistence ships only if precision clears.

The biggest systemic risk we name out loud: correlated detector failure. Until a genuinely independent second licensed detector exists, “m-of-n” is just n copies of one Google API — and a unanimous-wrong federation is unslashable. That’s why real federation is gated, not assumed.
Where it’s going

Criteria-gated, not calendar-driven

PHASE A

Testnet MVP

Single oracle on Base Sepolia + EAS, image + text. Graduates on the precision gate, a scoped audit, and a ≥4-week attack-net.

PHASE B

Guarded mainnet

L2 deployment; real token only if the kill-gates clear. Graduates on ≥3 paying integrators + HSM custody + a funded bounty.

PHASE C

Real federation

BLS m-of-n, gated on ≥2 genuinely independent licensed detectors and antitrust sign-off.

PHASE D

Decentralization

On-chain governance, token DVM, reputation-gated voting. Solana only if a capacity model justifies it.

Open source

Built in public, governed in two tracks

Code authority (a Technical Steering Committee) is deliberately separate from on-chain protocol governance (the DAO). Standards evolve as Juno Improvement Proposals. Contributions welcome.
